# MVP Definition

MVP определяется через правило из `Master_Prompt_v1.0`.

Функция входит в MVP, если помогает пользователю:

- получить смету;
- получить цены;
- получить объемы;
- получить подрядчика;
- получить поставщика;
- контролировать стройку.

Если функция не помогает ни одному из этих сценариев, она не входит в MVP и
должна быть перенесена в `docs/Backlog.md` или отложена до будущих версий.

## Текущий MVP-фокус

1. Пользователи, роли и доступы.
2. Поставщики и их аккаунты.
3. Загрузка прайсов поставщиков.
4. Сырые строки товаров из прайсов.
5. Нормализация материалов через модерацию.
6. Предложения поставщиков и история цен.
