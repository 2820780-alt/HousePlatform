# MODULE_01_MATERIAL_HUB_v2.0

Дата фиксации: 20.06.2026

Статус: архитектура v2.0 принята к реализации безопасным миграционным слоем.

## Назначение

Material Hub является фундаментом платформы АТОМ.

Это не каталог товаров и не агрегатор цен. Material Hub является единым центром данных о строительных материалах, поставщиках, производителях, ценах, документации и подготовленных знаниях.

Данные Material Hub используют:

- Module_02 Knowledge Base;
- сметный модуль;
- модуль проектирования;
- тендеры;
- маркетплейс;
- AI-помощник;
- аналитика.

## Главный объект

Все данные строятся вокруг сущности `Material`.

Связанные сущности:

- цены;
- поставщики;
- документы и ресурсы знаний;
- характеристики;
- аналоги;
- технологии-кандидаты;
- нормативы-кандидаты;
- история изменений;
- источники наполнения.

`Material` остается каноническим материалом и будущим источником для смет.

`CatalogProduct` остается карточкой товара из внешнего источника и не заменяет `Material`.

## Источники

Поддерживаемые типы источников:

- `RETAIL`;
- `MANUFACTURER`;
- `SUPPLIER`;
- `MANUAL_UPLOAD`.

Retail используется для ассортимента, цен, наличия и характеристик.

Производители используются для эталонных характеристик, документации, сертификатов, технологий применения, инструкций и типовых узлов.

## Логика наполнения

Целевая цепочка:

```text
Retail + Manufacturer
↓
CatalogProduct
↓
Material
↓
KnowledgeResource
↓
KnowledgeCandidate
↓
Module_02 Knowledge Base
```

Файлы поставщиков проходят отдельную цепочку:

```text
CSV/XLSX
↓
SupplierUpload
↓
SupplierUploadRow
↓
Material
↓
SupplierPrice
↓
PriceHistory
```

## Иерархия классификации

Используется структура:

```text
ConstructionGroup
↓
Category
↓
Subcategory
↓
MaterialType
↓
Material
```

`ConstructionGroup` является классификацией поверх категорий материалов и показывает часть дома или направление работ.

Примеры групп:

- Фундамент;
- Стены;
- Перегородки;
- Перекрытия;
- Кровля;
- Фасад;
- Гидроизоляция;
- Теплоизоляция;
- Звукоизоляция;
- Электрика;
- Отопление;
- Водоснабжение;
- Канализация;
- Внутренняя отделка;
- Наружная отделка;
- Крепеж;
- Инструмент.

## Автоклассификация

Для автоклассификации используется `ClassificationRule`.

Правило может учитывать:

- название;
- характеристики;
- бренд;
- производителя;
- ключевые слова;
- документы;
- категорию источника;
- раздел сайта.

Если уверенность низкая, материал попадает в очередь `Требует классификации`.

После ручной классификации администратор может создать правило, чтобы аналогичные материалы дальше классифицировались автоматически.

## Карточка Material

Карточка материала является главным экраном материала.

Вкладки:

- Общие данные;
- Характеристики;
- Цены;
- История цен;
- Поставщики;
- Документы;
- Технологии;
- Узлы;
- Нормативы;
- Аналоги;
- Конвертация единиц;
- Журнал изменений;
- Источники.

## KnowledgeResource

`KnowledgeResource` является новым v2.0-слоем для документов, ссылок и ресурсов знаний.

`MaterialDocument` сохраняется для обратной совместимости текущего API, задач анализа и уже созданных данных.

Внешние PDF не сохраняются локально. Хранятся:

- ссылка;
- метаданные;
- связи;
- извлеченные данные;
- при необходимости извлеченный текст.

Типы ресурсов:

- `CERTIFICATE`;
- `DECLARATION`;
- `PASSPORT`;
- `TECH_CARD`;
- `INSTALLATION_GUIDE`;
- `ARTICLE`;
- `VIDEO`;
- `TYPICAL_NODE`;
- `ALBUM`;
- `SP`;
- `SNIP`;
- `GOST`;
- `BIM`.

Один ресурс может быть связан с:

- `Material`;
- `ConstructionGroup`;
- `Category`;
- `Subcategory`;
- `MaterialType`;
- `Manufacturer`.

## KnowledgeCandidate

`KnowledgeCandidate` является промежуточным слоем между ресурсом и будущей базой знаний.

Типы кандидатов:

- `TECHNOLOGY`;
- `CONSUMPTION_RATE`;
- `APPLICATION_GUIDE`;
- `COMPATIBILITY_RULE`;
- `TYPICAL_NODE`;
- `INSTALLATION_STEP`;
- `MATERIAL_PROPERTY`.

Кандидат не является истиной автоматически.

Статусы:

- `AUTO_EXTRACTED`;
- `NEEDS_REVIEW`;
- `VERIFIED`;
- `REJECTED`.

## Аналоги

Сущность `MaterialAnalog` хранит связи между материалами.

Разделение:

- `ALTERNATIVE` — тот же материал с теми же характеристиками и брендом, но у другого поставщика;
- `ANALOG` — сопоставимый материал другого производителя или бренда;
- `REPLACEMENT` — замена, требующая отдельной проверки.

Финальное решение по аналогам принимает администратор.

## Конвертация единиц

Конвертация выполняется через `UnitConversionRule`.

Правило может быть привязано к:

- `Material`;
- `ConstructionGroup`;
- `Category`;
- `Subcategory`;
- `MaterialType`.

Конвертация выполняется только при наличии подтвержденных характеристик материала.

Если данных недостаточно, материал попадает в очередь `Требует данных для конвертации`.

## Очереди качества данных

Для контроля качества используется `MaterialQualityIssue`.

Типовые очереди:

- `CLASSIFICATION_REQUIRED`;
- `DUPLICATE_REVIEW_REQUIRED`;
- `DOCUMENTS_REQUIRED`;
- `SPECIFICATIONS_REQUIRED`;
- `CONVERSION_DATA_REQUIRED`;
- `TECHNOLOGY_REVIEW_REQUIRED`.

Очередь не удаляет материал и не меняет его автоматически.

## Задачи анализа

Журнал задач остается на существующих сущностях:

- `SourceTask`;
- `SourceTaskLog`;
- `SourceTaskResult`.

Текущие action-типы сохраняются для совместимости.

В терминах v2.0:

- `UPDATE_SPECIFICATIONS` соответствует обновлению характеристик;
- `UPDATE_TECH_DOCUMENTS` соответствует обновлению технических документов;
- `SCAN_TECHNOLOGIES` готовит кандидатов знаний;
- `UPLOAD_SUPPLIER_FILE` соответствует ручной загрузке поставщика.

## Что не делать в Module_01

Material Hub не должен:

- строить сметы;
- рассчитывать дома;
- создавать проекты;
- делать AI-выводы без данных;
- хранить внешние PDF-файлы локально;
- превращать кандидаты знаний в истину без проверки;
- заменять `Material` на `CatalogProduct`.

## Результат

Material Hub должен стать единственным источником достоверных данных о строительных материалах и подготовить структурированную основу для Module_02 Knowledge Base.
