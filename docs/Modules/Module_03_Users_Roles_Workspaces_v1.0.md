# Module_03_Users_Roles_Workspaces_v1.0

Дата фиксации: 20.06.2026

Статус: архитектурный фундамент доступа принят.

## Назначение

Module_03 отвечает за:

- пользователей;
- роли;
- права доступа;
- рабочие пространства;
- персональные дашборды;
- аудит действий;
- безопасность платформы.

Модуль является фундаментом всей системы доступа.

## Главный принцип

Не существует единого интерфейса для всех.

Каждый пользователь должен видеть только то, что относится к его роли, правам и рабочему пространству.

Схема:

```text
User
↓
Role
↓
Permission
↓
Workspace
↓
DashboardProfile
↓
Modules
↓
Functions
```

## Сущности

Основные сущности:

- `User`;
- `Role`;
- `Permission`;
- `RolePermission`;
- `UserRoleAssignment`;
- `Workspace`;
- `WorkspaceMember`;
- `DashboardProfile`;
- `DashboardWidget`;
- `DashboardWidgetPlacement`;
- `FavoriteModule`;
- `ModuleAccess`;
- `FunctionAccess`;
- `UserSession`;
- `UserPreference`;
- `RoleTemplate`;
- `AuditLog`.

`AuditEvent` сохраняется как существующий системный журнал. `AuditLog` является расширенным журналом Module_03 для действий пользователей и безопасности.

## Пользователь

Для пользователя хранить:

- id;
- email;
- телефон;
- имя;
- фамилию;
- статус;
- дату регистрации;
- последний вход;
- активность;
- совместимую системную роль;
- настраиваемые роли;
- рабочие пространства;
- настройки.

Поле `User.role` остается для обратной совместимости текущих API.

## Роли

Роли должны быть настраиваемыми.

Базовый справочник:

- `ADMIN`;
- `MANAGER`;
- `ESTIMATOR`;
- `ENGINEER`;
- `SUPPLIER`;
- `CONTRACTOR`;
- `CUSTOMER`;
- `ANALYST`;
- `MODERATOR`;
- `VIEWER`.

Администратор сможет создавать новые роли без изменения кода.

## Права

Права назначаются отдельно от ролей.

Примеры:

- `VIEW_MATERIALS`;
- `EDIT_MATERIALS`;
- `DELETE_MATERIALS`;
- `VIEW_DOCUMENTS`;
- `EDIT_DOCUMENTS`;
- `VIEW_PRICES`;
- `EDIT_PRICES`;
- `VIEW_USERS`;
- `MANAGE_USERS`;
- `VIEW_AUDIT`;
- `VIEW_KNOWLEDGE`;
- `EDIT_KNOWLEDGE`.

## Доступ к модулям

`ModuleAccess` задает уровень доступа:

- `NO_ACCESS`;
- `VIEW`;
- `EDIT`;
- `ADMIN`.

Доступ может быть назначен роли, пользователю или рабочему пространству.

## Доступ к функциям

`FunctionAccess` задает права внутри модуля.

Пример:

пользователь может смотреть материалы, но не может редактировать или удалять.

## Рабочие пространства

`Workspace` определяет:

- набор модулей;
- набор виджетов;
- оформление рабочего экрана;
- стартовую страницу;
- состав пользователей.

Примеры рабочих пространств:

- Администрирование;
- Сметный отдел;
- Закупки;
- Проектирование;
- Поставщик;
- Подрядчик;
- Заказчик.

## Персональные дашборды

Каждый пользователь должен иметь собственный дашборд.

Структура:

```text
DashboardProfile
↓
DashboardWidgetPlacement
↓
DashboardWidget
```

Пользователь сможет:

- добавлять виджет;
- удалять виджет;
- менять размер;
- менять порядок;
- скрывать виджет;
- закреплять виджет.

## Избранные модули

Пользователь может выбрать избранные модули.

Ограничение главного атомного экрана: отображать максимум 8 избранных модулей.

Остальные доступны через раздел "Все модули".

## Аудит и безопасность

`AuditLog` фиксирует:

- вход;
- выход;
- создание;
- редактирование;
- удаление;
- экспорт;
- импорт;
- изменение прав;
- изменение ролей;
- запуск анализа;
- загрузку файлов.

`UserSession` хранит:

- активные сессии;
- IP;
- user-agent;
- статус;
- время начала;
- последнюю активность;
- завершение.

## Связь с дашбордом АТОМ

Атомный экран должен получать данные из Module_03.

Роль пользователя определяет:

- какие модули отображать;
- какие виджеты показывать;
- какие данные доступны;
- какие действия разрешены.

## Что не делать

Не хардкодить роли.

Не хардкодить права.

Не делать один общий дашборд.

Не связывать права только с интерфейсом.

Не давать доступ к данным через скрытие кнопок.

Все проверки доступа должны выполняться на сервере.

## Результат

После реализации Module_03 платформа должна поддерживать:

- множество ролей;
- множество рабочих пространств;
- персональные дашборды;
- управление доступом;
- аудит действий;
- историю пользователей;
- безопасное разграничение данных и функций.
