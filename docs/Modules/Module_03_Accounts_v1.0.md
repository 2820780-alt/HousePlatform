# Module_03_Accounts_v1.0

## Цель

Управление пользователями, ролями, поставщиками и доступами.

## Бизнес-логика

- Пользователь может иметь роль администратора, поставщика, подрядчика, заказчика или наблюдателя.
- Аккаунт поставщика связывается с карточкой поставщика.

## Сущности

- User
- Supplier
- SupplierAccount
- SupplierBranch
- AuditEvent

## Ограничения

- Доступ к данным поставщика ограничен связанным аккаунтом.
- Административные действия должны попадать в аудит.

## Будущие доработки

- Приглашения пользователей.
- Гибкие права доступа.
