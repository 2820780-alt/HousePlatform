# Module_10_Marketplace_v1.0

## Цель

Маркетплейс поставщиков, подрядчиков и партнерских предложений.

## Бизнес-логика

- Маркетплейс строится поверх проверенной базы данных платформы.
- Коммерческие предложения связываются с реальными сущностями: поставщиками, материалами, услугами и регионами.

## Сущности

- MarketplaceListing
- PartnerOffer
- ContractorProfile
- SupplierProfile

## Ограничения

- Коммерческие данные должны быть проверяемыми.
- Маркетплейс не должен идти раньше базовых данных и алгоритмов.

## Будущие доработки

- Рейтинги и отзывы.
- Платные размещения.
