# Material Hub

Дата фиксации: 18.06.2026

Статус: принята новая целевая версия модуля.

## Главная цель

Модуль 1 создает каноническую базу строительных материалов.

Это не база цен и не каталог поставщиков. Это центральный Material Hub
платформы, на который дальше опираются:

- база знаний технологий;
- конструктор дома;
- генератор смет;
- проверка смет;
- закупки;
- тендеры;
- маркетплейс;
- аналитика;
- AI-консультант.

## Главный принцип

Сначала формируется собственная база материалов. Поставщики подключаются позже
и не создают каталог с нуля.

Правильная логика:

```text
Производители + retail-сайты
        ↓
Канонический Material
        ↓
Характеристики
        ↓
Документы
        ↓
Сертификаты
        ↓
Техническая документация
        ↓
Поставщики
        ↓
Цены
        ↓
История цен
```

Поставщик приходит в готовую экосистему и сопоставляет свои позиции с
существующими `Material`.

## Источники данных

### Производители

Производители являются источником эталонных данных:

- точные названия материалов;
- характеристики;
- размеры;
- сертификаты;
- декларации;
- паспорта качества;
- технические карты;
- инструкции;
- альбомы решений;
- типовые узлы;
- BIM-модели;
- рекомендации по применению.

### Retail-сайты

Retail используется для ассортимента, карточек товаров, цен и наличия:

- названия материалов;
- категории;
- характеристики;
- бренды;
- цены;
- наличие;
- регионы;
- ссылки на карточки товаров.

### Поставщики

Поставщик не создает общий материал напрямую. Он загружает свои позиции, а
система сопоставляет их с существующими `Material`.

Если совпадения нет, создается кандидат на новый `Material` и отправляется на
модерацию.

### Ручная загрузка файлов

MVP поддерживает только:

- CSV;
- XLSX.

Позже можно добавить:

- читаемые PDF;
- счета;
- КП;
- прайсы в PDF.

Сканы и фото не входят в MVP.

## Подсистема анализа сайтов

Администратор сначала выбирает источники, потом выбирает конкретное действие.
Нет единой кнопки "собрать все".

### Source

Для каждого источника хранится:

- название;
- тип источника;
- URL;
- статус;
- приоритет;
- дата последнего полного анализа;
- дата последнего обновления цен;
- дата последнего обновления сертификатов и документов;
- дата последнего анализа технологий.

Типы источников:

- `MANUFACTURER`;
- `RETAIL`;
- `SUPPLIER`;
- `MANUAL_UPLOAD`.

Статусы:

- `ACTIVE`;
- `PAUSED`;
- `DISABLED`;
- `ERROR`.

### Действия источников

Каждое действие выполняет только свою задачу:

- `INITIAL_MATERIAL_SCAN` — первичное формирование базы материалов;
- `UPDATE_PRICES` — обновление цен и `PriceHistory`;
- `FIND_NEW_PRODUCTS` — поиск новых товаров;
- `UPDATE_SPECIFICATIONS` — обновление характеристик;
- `UPDATE_CERTIFICATES` — обновление сертификатов;
- `UPDATE_TECH_DOCUMENTS` — обновление технической документации;
- `SCAN_TECHNOLOGIES` — подготовка кандидатов для Knowledge Base;
- `FULL_INITIAL_SCAN` — редкая полная первичная загрузка источника;
- `CHECK_SOURCE_HEALTH` — проверка доступности источника;
- `UPLOAD_SUPPLIER_FILE` — загрузка CSV/XLSX поставщика или администратора.

Если выбрано `UPDATE_PRICES`, система обновляет только цены.
Если выбрано `UPDATE_CERTIFICATES`, система проверяет только сертификаты.
Если выбрано `SCAN_TECHNOLOGIES`, система ищет только технологии и данные для
Knowledge Base.

Лишние процессы без команды администратора не запускаются.

## Ручная загрузка

Файл не создает отдельный каталог.

Цепочка обработки:

```text
Прайс / счет / КП
        ↓
Строки файла
        ↓
Нормализация
        ↓
Поиск существующего Material
        ↓
Сопоставление
        ↓
Обновление цены поставщика
        ↓
История цен
```

Если `Material` не найден:

```text
Новая позиция
        ↓
Кандидат на Material
        ↓
Модерация
        ↓
Создание Material при подтверждении
```

## Поиск дублей

Главное правило:

```text
Сначала категория
↓
Потом характеристики
↓
Потом поиск дублей
```

Нельзя сравнивать все материалы между собой.

Алгоритм:

1. Определить категорию.
2. Проверить совместимость категорий.
3. Выбрать шаблон характеристик.
4. Сравнить релевантные характеристики.
5. Рассчитать `match_score`.
6. Очевидные совпадения связать автоматически.
7. Спорные совпадения отправить в `MaterialMatchCandidate`.
8. При необходимости использовать AI как помощника.
9. Финальное решение принимает администратор.

AI не принимает окончательное решение.

## Роль администратора

Администратор:

- добавляет источники;
- выбирает сайты;
- выбирает действие;
- запускает анализ;
- смотрит результаты;
- проверяет спорные совпадения;
- подтверждает новые `Material`;
- отклоняет мусорные позиции;
- контролирует качество данных;
- может загрузить CSV/XLSX вручную.

Администратор не занимается массовым ручным созданием базы.

## Роль поставщика

Поставщик в будущем через личный кабинет может:

- загрузить CSV/XLSX прайс;
- загрузить остатки;
- обновить цены;
- увидеть сопоставление своих позиций с `Material`;
- исправить спорные строки;
- предложить новый `Material`.

Поставщик не создает общий `Material` напрямую.

## Связь с Knowledge Base

Material Hub передает в Knowledge Base:

- материалы;
- категории;
- производителей;
- характеристики;
- сертификаты;
- технические карты;
- инструкции;
- типовые узлы;
- альбомы решений;
- BIM;
- рекомендации производителей;
- найденные технологии;
- найденные нормы расхода.

Knowledge Base строится поверх Material Hub и не должна быть отдельной
абстрактной базой без связи с реальными материалами.

## MVP

В MVP входят:

- справочник источников;
- выбор одного, нескольких или всех источников;
- выбор действия;
- очередь задач `SourceTask`;
- логи задач `SourceTaskLog`;
- результаты задач `SourceTaskResult`;
- Material Hub сущности;
- CSV/XLSX импорт;
- первичное сопоставление дублей;
- модерация спорных случаев;
- `UPDATE_PRICES`;
- `INITIAL_MATERIAL_SCAN`;
- `UPLOAD_SUPPLIER_FILE`.

## Не входит в MVP

- сложный обход всех сайтов;
- антибот-обходы;
- полноценный AI;
- обработка сканов;
- обработка фото;
- глубокий анализ любых PDF;
- автоматическое принятие спорных решений;
- маркетплейс;
- личный кабинет поставщика в полной версии;
- автоматическое создание норм расхода без проверки.
