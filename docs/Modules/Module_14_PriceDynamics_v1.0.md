# Module_14_PriceDynamics_v1.0

# История и динамика цен

Дата фиксации: 19.06.2026

Статус: целевая версия модуля сформирована.

## Главная цель

Модуль 14 анализирует изменение цен и наличия строительных материалов во
времени.

Модуль не собирает данные самостоятельно. Источником фактов является
`Module_01_Materials`:

- `Material`;
- `MaterialCategory`;
- `CatalogProduct`;
- `Source`;
- `SupplierPrice`;
- `PriceHistory`.

`PriceHistory` остается фактическим журналом в Material Hub. Модуль 14 строит
поверх него представления, фильтры, графики и аналитические срезы.

## Бизнес-логика

Модуль отвечает на вопросы:

- как изменилась цена конкретного материала;
- как изменилась цена по категории материалов;
- как изменилась цена по производителю;
- как изменилась цена по региону;
- как изменилась цена у конкретного источника или поставщика;
- есть ли изменение наличия во времени;
- какие материалы подорожали или подешевели сильнее всего.

## MVP

В MVP нужно реализовать:

- таблицу истории цен с фильтрами;
- фильтр по периоду;
- фильтр по материалу;
- фильтр по категории;
- фильтр по производителю;
- фильтр по региону;
- фильтр по источнику;
- отображение цены;
- отображение изменения цены в рублях;
- отображение изменения цены в процентах;
- график цены конкретного материала;
- график изменения категории материалов в процентах;
- отдельный признак наличия, если источник его предоставляет.

## Стартовая реализация

Модуль можно запускать параллельно с Material Hub.

Первый экран модуля:

- читает данные из `PriceHistory`;
- показывает количество точек истории;
- показывает материалы с историей и без истории;
- показывает изменение последней цены относительно предыдущей;
- показывает изменение в рублях и процентах;
- показывает график изменения цены конкретного материала;
- фильтрует данные по материалу, категории, производителю, региону и источнику.

## Граница с Модулем 1

Модуль 1 собирает, обрабатывает и хранит данные.

Модуль 14 анализирует историю изменения цен.

Аналитика изменения цен не должна находиться на панели управления Модуля 1.

## Что не входит в MVP

Не делать сразу:

- прогнозирование цен;
- AI-объяснение причин изменения цены;
- автоматические финансовые рекомендации;
- закупочные решения без подтверждения человека;
- сложные рыночные индексы без достаточной истории;
- сравнение с внешними макроэкономическими данными.

## Основные срезы

### Общий индекс

Показывает усредненное изменение цен по всем материалам, по которым есть
история.

Используется только как ориентир. Не является финансовым прогнозом.

### Категория материалов

Пример:

- кровельные материалы;
- стеновые материалы;
- сухие смеси;
- пиломатериалы;
- крепеж;
- электрика;
- сантехника.

### Конкретный материал

Показывает:

- цену во времени;
- изменение в рублях;
- изменение в процентах;
- источники цены;
- регион;
- наличие, если данные есть.

### Производитель

Показывает динамику по материалам конкретного производителя.

### Регион

Показывает динамику по городу или региону, если источник передает регион.

## Связь со строительными группами

Модуль 14 должен уметь фильтровать данные по строительным группам из
`Module_15_ConstructionGroups`.

Пример:

```text
Фундамент
  ↓
бетон
крепеж
арматура
пиломатериал для опалубки
гидроизоляция
```

Наполнение групп выполняется не в Модуле 14. Модуль 14 только использует
готовую классификацию для аналитических срезов.

## Сущности

Для MVP можно обойтись без новых обязательных таблиц и строить аналитику по
`PriceHistory`.

Будущие сущности:

- PriceAnalyticsView;
- SavedPriceFilter;
- PriceIndexSnapshot;
- AvailabilitySnapshot.

## Ограничения

- Модуль 14 не изменяет `Material`.
- Модуль 14 не изменяет цены.
- Модуль 14 не принимает закупочных решений.
- Модуль 14 не является источником истины.
- Все фактические цены приходят из Material Hub.
