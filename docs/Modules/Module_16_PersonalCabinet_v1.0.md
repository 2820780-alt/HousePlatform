# Module_16_PersonalCabinet_v1.0

# Личный кабинет

Дата фиксации: 19.06.2026

Статус: целевая версия модуля сформирована.

## Главная цель

Модуль 16 является пользовательской оболочкой платформы.

Он показывает пользователю доступные модули, уведомления, рабочие карточки и
быстрые действия.

Личный кабинет не должен хранить доменные данные других модулей.

## Принцип

Личный кабинет строится как модульная панель.

Каждая карточка соответствует модулю или рабочему разделу:

- Material Hub;
- История цен;
- Документы;
- Сметы;
- Закупки;
- Тендеры;
- Поставщики;
- Аналитика;
- Настройки аккаунта.

Пользователь может скрывать или показывать карточки, если это не нарушает его
роль и права доступа.

## Связь с аккаунтами

Модуль 16 использует `Module_03_Accounts` для:

- пользователя;
- роли;
- прав доступа;
- связи с поставщиком;
- связи с заказчиком или подрядчиком.

`Module_03_Accounts` отвечает за доступы. `Module_16_PersonalCabinet` отвечает
за отображение и навигацию.

## MVP

В MVP нужно реализовать:

- главную панель пользователя;
- список доступных модулей;
- карточки модулей;
- счетчики уведомлений;
- быстрые действия;
- скрытие и показ карточек;
- сохранение пользовательского порядка карточек;
- разные наборы карточек для администратора, поставщика и заказчика.

## Что не входит в MVP

Не делать сразу:

- сложный конструктор интерфейса;
- пользовательские виджеты произвольного типа;
- AI-персонализацию;
- финансовые рекомендации;
- маркетплейс внутри личного кабинета.

## Сущности

- UserModulePreference;
- DashboardCardPreference;
- ModuleNotification;
- QuickAction.

## Ограничения

- Личный кабинет не меняет архитектуру доменных модулей.
- Скрытие карточки не отключает сам модуль.
- Доступ к модулю определяется правами, а не видимостью карточки.
- Карточки должны показывать только данные своего модуля или агрегированные
  счетчики через публичный API модуля.
