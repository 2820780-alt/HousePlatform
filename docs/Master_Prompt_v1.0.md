## Назначение файлов

### Master_Prompt

Содержит только:

- цели проекта;
- принципы;
- архитектуру;
- очередность модулей;
- ограничения;
- стратегические решения.

Редактируется редко.

Является главным документом проекта.

### Файлы модулей

Каждый модуль содержит:

- цель;
- бизнес-логику;
- сущности;
- алгоритмы;
- ограничения;
- будущие доработки;
- историю изменений.

### Roadmap

Содержит только:

- этапы разработки;
- порядок реализации;
- MVP;
- следующие версии;
- приоритеты.

Roadmap может меняться часто.

### Decisions

Содержит все архитектурные решения.

Каждое решение имеет:

- номер;
- дату;
- описание;
- причину принятия;
- последствия.

## Правила изменения модулей

Любое изменение оформляется как новая версия.

Пример:

```text
Module_01_Materials_v1.0
↓
Module_01_Materials_v1.1
```

Если изменение небольшое: `v1.0` → `v1.1`.

Если изменение крупное: `v1.0` → `v2.0`.

## Правило работы с ChatGPT и Claude

При обсуждении использовать команды:

- "Редактируем Модуль 1"
- "Создаем версию 1.1 Модуля 2"
- "Добавляем решение в Decisions"
- "Обновляем Roadmap"

## Правило защиты архитектуры

Новая идея не должна сразу попадать в проект.

Для каждой идеи необходимо ответить:

1. Какую проблему решает?
2. Какой модуль изменяет?
3. Какие данные собирает?
4. Можно ли решить проще?
5. Можно ли решить без AI?
6. Нужна ли она для MVP?
7. Повышает ли коммерческую ценность?

Если ответы неочевидны — идея отправляется в Backlog.

## Правило MVP

Любая функция сначала проверяется вопросом:

Помогает ли она пользователю:

- получить смету;
- получить цены;
- получить объемы;
- получить подрядчика;
- получить поставщика;
- контролировать стройку.

Если нет — функция не входит в MVP.

## Правило AI

AI никогда не является источником истины.

AI может:

- помогать искать;
- объяснять;
- анализировать;
- классифицировать;
- предлагать варианты.

AI не может:

- самостоятельно создавать нормы;
- самостоятельно принимать финансовые решения;
- самостоятельно объединять спорные данные без подтверждения.

## Правило развития платформы

Порядок важнее количества функций.

Последовательность развития:

```text
данные
↓
алгоритмы
↓
аналитика
↓
автоматизация
↓
AI
```

Никогда наоборот.

## Текущее состояние проекта

Зафиксировано как: `Master_Prompt_v1.0`.

Следующая работа выполняется только через изменение отдельных модулей и запись решений в `Decisions.md`.

С этого момента:

- `Master_Prompt_v1.0` не трогаем без серьезной причины;
- любые изменения делаем через команду "Редактируем Модуль X";
- любые важные договоренности заносим в `Decisions.md`;
- все спорные идеи сначала попадают в Backlog, а не сразу в архитектуру.
