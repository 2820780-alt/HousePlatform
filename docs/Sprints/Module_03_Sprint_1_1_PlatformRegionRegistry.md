# Module 03 Sprint 1.1. PlatformRegionRegistry

Дата фиксации: 25.06.2026

Статус: реализован платформенный справочник регионов.

## Цель

Добавить platform registry регионов для multi-region-ready архитектуры АТОМ.

Регионы не являются отдельным бизнес-модулем. Это платформенный справочник,
который будут использовать:

- цены;
- поставщики;
- работы;
- сметы;
- закупки;
- marketplace;
- логистика;
- аналитика.

## Реализованные сущности

- `PlatformRegion`;
- `PlatformCity`;
- `DeliveryZone`;
- `ActiveRegion`;
- `PilotRegion`;
- `RegionStatus`.

## Стартовая запись MVP

Добавлена seed-запись:

```text
code = KRASNODAR_KRAI
name = Краснодарский край
country = RU
status = ACTIVE
isPilotRegion = true
isActive = true
```

Краснодарский край является записью registry, а не hardcoded условием в коде.

## Правило для кода

Запрещено:

```text
region = "Краснодарский край"
```

Правильно:

- `region_id`;
- `city_id`;
- `delivery_zone_id`;
- `price_region_id`;
- `source_region_id`;
- `supplier_region_id`;
- `work_region_id`.

## Будущие административные действия

Admin / Super Admin в будущем сможет:

- включить регион;
- отключить регион;
- сделать регион пилотным;
- открыть регион для пользователей;
- открыть регион для поставщиков;
- открыть регион для marketplace;
- открыть регион для аналитики;
- архивировать регион.

Обычный пользователь не управляет списком регионов платформы.

## Ограничения текущего спринта

В этом спринте не выполняется:

- перевод старых строковых полей `region` и `city` на registry;
- изменение Module 01, Module 11 или других бизнес-модулей;
- создание UI управления регионами;
- подключение реальной авторизации.
