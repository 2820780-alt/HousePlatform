# Sprint_10_UI_Standards_and_RBAC_Preparation

Дата фиксации: 20.06.2026

Статус: базовые UI-стандарты подключены к ключевым экранам.

## Цель

Унифицировать интерфейс платформы АТОМ без изменения бизнес-логики и структуры модулей.

Подготовить интерфейс к появлению:

- ролей;
- прав доступа;
- рабочих пространств;
- персональных дашбордов.

## Что сделано

Добавлен общий шаблон UI-стандартов:

```text
templates/_atom_ui_standards.html
```

Он задает:

- общие цвета АТОМ;
- базовые карточки;
- базовые панели;
- поведение таблиц;
- sticky-заголовки таблиц;
- единые empty states;
- RBAC-ready атрибуты для будущего управления видимостью.

## UX-исправления

Ключевые экраны переведены на единую основную прокрутку страницы.

Вертикальные внутренние scroll-контейнеры таблиц убраны там, где они мешали работе.

Таблицы получают горизонтальный scroll через единый контейнер:

```text
atom-table-wrap
```

## Подготовка к RBAC

Ключевые визуальные блоки получают атрибуты:

- `data-atom-page`;
- `data-rbac-ready`;
- `data-module-number`;
- `data-widget-key`;
- `data-permission-key` в будущих формах и действиях.

Эти атрибуты не являются проверкой безопасности.

Серверные проверки доступа должны выполняться в Module_03.

## Что не менялось

- бизнес-логика;
- маршруты;
- модели данных;
- задачи анализа;
- импорт;
- обработка цен;
- структура модулей;
- атомная концепция главного экрана.

## Следующий шаг

После стабилизации Module_03 перевести реальные серверные проверки доступа на:

- `Role`;
- `Permission`;
- `ModuleAccess`;
- `FunctionAccess`.
