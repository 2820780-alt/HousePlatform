## Правило интеграций

Интеграция источника должна быть отдельным слоем над доменной моделью.

Интеграция может:

- проверять доступность источника;
- читать публичные страницы и документы;
- создавать и обновлять `CatalogProduct`;
- обновлять цены;
- писать `PriceHistory`;
- создавать `Material` только как часть контролируемого сценария Material Hub;
- создавать `MaterialMatchCandidate` для спорных случаев;
- сохранять документы без глубокого анализа содержимого.

Интеграция не может:

- обходить антибот-защиту;
- использовать приватные API без отдельного решения;
- принимать спорные решения вместо администратора;
- создавать нормы расхода;
- менять Knowledge Base.
