Domain Model HousePlatform
Главная идея доменной модели

HousePlatform строится вокруг объекта строительства.

Объект строительства связывает пользователя, регион, проект дома, технологии, материалы, сметы, закупки, поставщиков, подрядчиков, документы, фото, события и будущий цифровой двойник.

Строительный объект

Строительный объект — главный центр данных платформы.

эксплуатационные данные;
цифровой двойник дома.
Базовые сущности MVP
Пользователь

Хранит данные владельца аккаунта.

Связи:

может иметь несколько строительных объектов;
может загружать документы;
может создавать сметы;
может запускать AI-анализ.
Регион

Нужен для региональных цен, нормативов, доставки, климата и доступности материалов.

Связи:

связан с объектом строительства;
связан с поставщиками;
связан с ценами;
связан с нормативами.
Проект дома

Описание проектного решения дома.

Связи:

связан с объектом строительства;
связан с технологиями;
связан с расчетами;
связан с документами.
Технология строительства

Например:

газобетон;
каркас;
кирпич;
монолит;
СИП;
брус.

Связи:

связана с материалами;
связана с нормативами;
связана с расчетами;
связана с правилами совместимости.
Материал

Базовая сущность для расчетов, смет, закупок и AI-анализа.

Связи:

категория материала;

единицы измерения;
бренд;
производитель;
поставщики;
цены;
история цен;

Поставщик

Организация или источник поставки материалов.

Связи:

материал;
поставщик;
регион;
склад;

Хранит изменение цены во времени.

Нужна для:

Главное правило

Ни одна важная сущность не должна существовать отдельно от общей модели данных.

Все данные должны быть связаны с будущими расчетами, аналитикой, AI, маркетплейсом и цифровым двойником дома.
