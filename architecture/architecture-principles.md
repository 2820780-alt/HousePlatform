Architecture Principles HousePlatform
1. Данные важнее интерфейса

Интерфейс может меняться много раз. Данные должны проектироваться так, чтобы жить годами.

2. Объект строительства — центральная сущность

Нельзя проектировать сметы, закупки, фото, документы или AI-анализ отдельно от объекта строительства.

Все ключевые данные должны быть связаны с объектом.

3. Нельзя проектировать этапы изолированно

Каждый этап должен учитывать:

существующие сущности;
новые сущности;
связи;
историю;
события;
будущую аналитику;
будущий AI;
маркетплейс;
цифровой двойник дома.
4. История изменений обязательна

Для важных данных нужно хранить историю:

цены;
сметы;
расчетные параметры;
статусы;
документы;
события;
решения пользователя.
5. События обязательны

Система должна логировать важные действия:

создание объекта;
изменение сметы;
загрузка документа;
изменение цены;
добавление фото;
AI-анализ;
изменение статуса этапа строительства.
6. AI работает поверх данных платформы

AI не должен быть отдельной игрушкой.

Любой AI-модуль должен:

использовать данные платформы;
сохранять результат анализа;
объяснять источник вывода;
создавать события;
улучшать базу знаний.
7. Расчеты должны быть версионируемыми

Каждый расчет должен иметь:

входные параметры;
формулы;
нормативы;
коэффициенты;
региональные поправки;
источник данных;
версию расчета;
историю изменений.
8. Система должна быть переносимой

Проект нельзя жестко привязывать к одному сервису, хостингу или AI-инструменту.

Должна быть возможность перенести проект на другой сервер.

9. Бэкапы обязательны

Резервное копирование должно включать:

код;
базу данных;
документы;
фото;
настройки;
инструкции восстановления.
10. Простота сопровождения важнее моды

При выборе технологий предпочтение отдавать решениям, которые:

популярны;
хорошо документированы;
поддерживаются сообществом;
понятны новичку при наличии инструкции;
не создают критической зависимости от одного поставщика.
